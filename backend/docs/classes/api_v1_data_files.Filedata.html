Protected prepareDocumentForUpload

  • prepareDocumentForUpload(user: AuthUser, files: any, doc: IFileMetadata, exists: boolean, map: Map<DataManager<any>, any[]>, lastDBId: string): Promise<IFileMetadata>

  • Inherited from DBManager.prepareDocumentForUpload

    • Defined in src/api/v1/DataManager.ts:402

    Converts a document of mixed objects and foreign keys into a database-safe object, ready for uploading. The final document (and any necessary sub documents) are added to map, so all documents can be added to the database at the same time. This avoids issues where an error during parsing would leave documents half-uploaded.

    This process is very expensive, especially for Projects, mostly due to the inefficiencies of the process due to keeping it generic. Thankfully, this is only used on administrative routes, where mixed data is allowed.

    Parameters

    • user: AuthUser

      The user that started the upload

    • files: any

      Any files with the upload

    • doc: IFileMetadata

      The document to prepare

    • exists: boolean

      True if this document exists in the database

    • map: Map<DataManager<any>, any[]>

      The map to add all documents to

    • lastDBId: string

      The last valid database ID. Used for setting parent fields of sub-ojects.

    Returns Promise<IFileMetadata>

    doc, but with all foreign keys as IDs
